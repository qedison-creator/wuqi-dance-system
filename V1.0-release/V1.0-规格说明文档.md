# 舞柒舞蹈小程序 V1.0 规格说明文档

> 版本：V1.0  
> 发布日期：2026-05-24  
> 项目类型：微信小程序 + Node.js 后端  
> 数据库：MongoDB (wuqi_dance)

---

## 一、项目概述

舞柒舞蹈小程序是一套完整的舞蹈工作室管理系统，包含三个子项目：

| 子项目 | 技术栈 | 用途 |
|--------|--------|------|
| wuqi-member | 微信小程序原生 | 会员端，供学员预约课程、查看教练等 |
| wuqi-admin | 微信小程序原生 | 管理端，供管理员管理会员、排课、运营等 |
| wuqi-backend | Node.js + Express + MongoDB | 后端 API 服务，处理业务逻辑与数据存储 |

---

## 二、系统架构

### 2.1 技术栈

```
后端:
  - Node.js + Express.js (HTTP 框架)
  - MongoDB + Mongoose (数据持久化)
  - JWT (身份认证)
  - Multer (文件上传)
  - node-cron (定时任务调度)
  - dayjs (日期处理)

会员端:
  - 微信小程序原生框架 (WXML + WXSS + JS)
  - Custom Tab Bar (自定义底部导航)
  - WXS (模板脚本)

管理端:
  - 微信小程序原生框架
  - Custom Tab Bar
```

### 2.2 数据流

```
[会员端小程序]  --- HTTP/HTTPS --->  [Express API Server]  --- Mongoose --->  [MongoDB]
[管理端小程序]  --- HTTP/HTTPS --->       :3000/api/v1                       wuqi_dance
```

### 2.3 文件上传架构

```
上传请求 (带 type 参数)
  → upload.routes.js (resolveSubDir 中间件)
  → 按类型自动存入对应子目录:
      type=coach_avatar → uploads/coaches/
      type=coach_album  → uploads/coaches/
      type=coach_video  → uploads/videos/
      type=course       → uploads/courses/
      type=user_avatar  → uploads/users/
      type=banner       → uploads/banners/
      type=general      → uploads/general/
  → 静态文件服务 (文件不存在返回占位图)
```

## 四、核心业务规则

### 4.1 用户状态流转

```
游客 → 微信授权登录 → 已注册(未审核) → 管理员审核 → 正式会员
                                              ↓
正式会员 → 管理员限制 → 黑名单(disabled)
正式会员 → 管理员解除限制 → 恢复正常
```

### 4.2 套餐状态管理

```
已购套餐 → 待激活(2个月自动激活) → 使用中 → 已过期/次卡耗尽
                  ↓                        ↓
            管理员手动激活           管理员停卡 → 已停卡 → 恢复停卡
                  ↓
            管理员延长有效期
```

### 4.3 假期处理规则 (N2)

1. 设置放假期间时：
   - 自动取消放假期间的所有已有预约
   - 所有使用中的套餐有效期顺延放假天数
   - 已停卡套餐的 suspend_end_date 也同步顺延放假天数
   
2. 撤销放假时：
   - 套餐有效期回滚（减去之前顺延的天数）
   - 已停卡套餐的 suspend_end_date 也同步回滚

### 4.4 预约人数可见性 (D1)

只有以下条件**全部满足**时，会员才能查看课程预约人数：
- 已登录（非游客）
- 用户状态为"正式会员"（已通过审核）
- 已录入套餐（含待激活状态）
- 套餐在使用中
- 未被停卡
- 套餐未过期
- 次卡未耗尽
- 未被拉入黑名单

### 4.5 黑名单机制 (M7/D5/D6)

- 管理员可将会员状态设置为 `disabled`（限制使用）
- 受限会员无法预约任何课程
- 受限会员不会收到微信订阅消息提醒

### 4.6 微信订阅消息通知 (D6)

13种通知场景：
1. 预约成功通知
2. 预约取消通知
3. 候补成功通知
4. 课程即将开始提醒
5. 套餐即将到期提醒
6. 套餐已到期通知
7. 套餐激活通知
8. 会员审核通过通知
9. 会员审核不通过通知
10. 转卡申请审核结果通知
11. 停卡通知
12. 恢复停卡通知
13. 黑名单限制通知

所有消息发送前都会过滤：disabled用户、已删除用户。

### 4.7 门店隔离

- 每个门店的数据相互独立
- 通过 storeFilter 中间件实现门店级数据隔离
- 管理端和会员端均支持门店切换
