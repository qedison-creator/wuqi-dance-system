# 公告弹窗切换交互重新设计 Spec

## Why
当前公告弹窗的切换导航按钮放置在弹窗卡片外部，视觉脱节且风格陈旧。需要借鉴现代移动端产品的交互范式，设计更优雅、更直觉的多内容切换体验。

## What Changes
- 采用 **Instagram Stories 风格的进度条** 作为页码指示和切换入口，替代传统圆点+箭头
- 所有交互元素收入卡片内部
- 关闭按钮移入卡片头部右上角（简洁 ✕）
- 底部仅保留日期信息
- 支持滑动手势切换（主交互）+ 点击进度条段跳转（次交互）

## 设计方案："Story Progress"

### 核心理念
借鉴 Instagram Stories、微信朋友圈广告等现代产品的进度条交互模式——用细长的分段进度条同时承担"指示当前位置"和"快速跳转"两个功能，无需额外的箭头按钮。

### 视觉布局

**多条公告时：**
```
┌─────────────────────────────────┐
│  🔔 公告                    ✕   │  ← 头部：图标+标签+关闭✕
│  ▮▮▮▯▯                         │  ← 进度条（5段，前3段已填充）
│                                  │
│  公告标题                        │  ← 内容区
│  ──                              │
│  公告正文内容...                  │
│                                  │
│  06/05周四                       │  ← 底部：仅日期
└─────────────────────────────────┘
```

**单条公告时：**
```
┌─────────────────────────────────┐
│  🔔 公告                    ✕   │  ← 头部：图标+标签+关闭✕
│                                  │  （无进度条）
│  公告标题                        │
│  ──                              │
│  公告正文内容...                  │
│                                  │
│  06/05周四                       │
└─────────────────────────────────┘
```

### 进度条设计细节

| 属性 | 值 | 说明 |
|------|-----|------|
| 高度 | 6rpx | 极细，不抢视觉 |
| 间距 | 6rpx | 段与段之间 |
| 圆角 | 3rpx | 微圆角 |
| 未填充色 | rgba(42,33,34,0.1) | 极浅灰 |
| 已填充色 | linear-gradient(90deg, #D4786E, #E8985E) | 品牌渐变 |
| 动画 | width transition 300ms ease | 切换时平滑填充 |
| 可点击 | 是 | 点击某段可跳转到对应公告 |

### 关闭按钮设计细节

| 属性 | 值 | 说明 |
|------|-----|------|
| 位置 | 头部右侧，与"公告"标签同行 | 卡片内部 |
| 样式 | 纯文字 ✕ 符号 | 无圆环、无背景 |
| 颜色 | #948284 | 柔和灰 |
| 字号 | 32rpx | 与标题同级 |
| 触摸区域 | 64rpx × 64rpx | 满足最小触摸目标 |
| 反馈 | opacity: 0.5 + scale(0.9) | 按下反馈 |

### 切换交互

1. **主交互：左右滑动** — 在内容区域左右滑动切换公告（微信小程序 touch 事件实现）
2. **次交互：点击进度条段** — 点击某个进度条段直接跳转到对应公告
3. **无箭头按钮** — 进度条本身已足够直觉，箭头按钮是冗余的

### 切换动画

- 内容区域使用 CSS transition 实现淡入淡出（opacity 300ms ease）
- 进度条段使用 width transition 实现平滑填充

## Impact
- Affected code: `index.wxml`（弹窗结构）、`index.wxss`（弹窗样式）、`index.js`（添加滑动手势处理）

## ADDED Requirements

### Requirement: Stories 风格进度条导航

系统 SHALL 在公告弹窗头部下方显示分段进度条，每段对应一条公告，已读公告段以品牌渐变色填充。

#### Scenario: 多条公告进度条显示
- **WHEN** 公告数量 > 1 且弹窗打开
- **THEN** 头部下方显示 N 段进度条，当前及之前的段为品牌渐变填充色，之后的段为浅灰色

#### Scenario: 点击进度条段跳转
- **WHEN** 用户点击某个进度条段
- **THEN** 切换到对应索引的公告，进度条同步更新填充状态

#### Scenario: 滑动切换公告
- **WHEN** 用户在内容区域向左滑动（快速左滑）
- **THEN** 切换到下一条公告（如有）
- **WHEN** 用户在内容区域向右滑动（快速右滑）
- **THEN** 切换到上一条公告（如有）

#### Scenario: 单条公告
- **WHEN** 公告数量 = 1
- **THEN** 不显示进度条，头部与内容区间无额外元素

### Requirement: 头部内关闭按钮

系统 SHALL 在弹窗卡片头部右上角显示关闭按钮（✕符号），点击关闭弹窗。

#### Scenario: 点击关闭按钮
- **WHEN** 用户点击头部右上角 ✕ 按钮
- **THEN** 弹窗关闭

## MODIFIED Requirements

### Requirement: 公告弹窗头部
头部区域包含：左侧（图标+标签）、右侧（关闭✕按钮）。移除页码文字显示。

### Requirement: 公告弹窗底部
底部仅保留日期信息，不再包含任何导航或关闭元素。

## REMOVED Requirements

### Requirement: 卡片外部导航行
**Reason**: 所有交互元素应在卡片内部
**Migration**: 删除 `am-nav-row` 及相关样式

### Requirement: 头部页码显示
**Reason**: 进度条已隐式传达位置信息，无需文字页码
**Migration**: 删除 `am-page` 元素

### Requirement: 底部圆点指示器
**Reason**: 进度条替代圆点指示器，更现代更直觉
**Migration**: 删除 `am-dots`、`am-dot` 元素，用进度条替代

### Requirement: 外部箭头按钮
**Reason**: 滑动手势+进度条点击已覆盖切换需求，箭头按钮冗余
**Migration**: 删除箭头按钮，用滑动手势替代
